信件网域摘要直接在 Outlook Copilot 跑：给网域与时间范围，就回一张表——谁寄的、何时寄的、需要什么行动、我方有没有回，把客户经理／PM 每天最怕漏接的事变成 1 分钟检查表。
🎁 你将获得什么？
🔎 锁定一个客户
只看指定网域的来信，避免被 100 封通知信淹没。
✅ 行动栏
每封信都标出对方期待你做什么——不再每天重读整条 thread。
↩️ 已回复栏
自动标示我方是否已有人回复，漏接客户来信再见。
🚀 直接复制这段 Prompt（简中版）
创建一张表格，列出我在过去 X 周内从 DOMAINNAME（例如 @customer.com）收到的所有 email。请包含一栏显示这封信需要哪些行动，以及另一栏显示我公司是否有人已经回复。
📋 Prompt Template (English)
Create a tableshowing all emails I received from DOMAINNAME (e.g., @customer.com) in the last X weeks. Include a column that shows any action required from the message and another column to show if anyone at my company has responded yet.
💡 三个典型使用场景
1
📧 在 Outlook 的 Copilot 打开
桌面或行动版皆可，直接粘贴 Prompt。
2
✏️ 换掉 DOMAINNAME 与 X 周
换成实际客户网域（例如 @chunghwa.com.tw）与要看的范围（1、2、4、8…周）。
3
🎯 先处理「尚无人回复」
那一栏就是你今天最该补的待办，已回的列再确认回得对不对。
