每次都在猜对方要什么
大多数消息失败，不是因为内容有问题——而是因为风格不符合阅读者的期待。负责人要的是决策，技术角色要的是精确度，其他人可能需要先有脉络。

这个 Prompt 让 Copilot 根据收件人过去 90 天在 Outlook 和 Teams 的真实交互模式，分析其沟通偏好，再把你的文本改写成最匹配的版本。不是猜测，是基于真实数据的风格适配——让你每封信、每则消息都能获得想要的回应。
🎁 你将获得什么？
🔍 收件人沟通风格侧写
语气 / 长度 / 结构 / 技术深度全面分析，含信心评级。
✍️ 三种长度改写版本
短版（≤6行）、标准版（≤150字）、负责人决策版一次拿到。
📌 Do / Don't 规则清单
最多 8 条，让你以后写信时也有依据。
💡 主旨行 + 追问问题
建议信件主旨，以及一个解除下一步卡点的追问。
🚀 直接复制这段 Prompt（简中版）
收件人：[姓名 / 身份]
输入文本：[在此粘贴你的邮件 / 文本 / 摘要]

目标：改写输入文本，使其符合收件人的沟通风格与角色场景（技术型 / 非技术型），同时保持专业、准确、基于事实。

数据源（仅限 Work 模式）：使用过去 90 天与收件人的 Outlook 邮件和 Teams 消息，以及可见的个人数据信号（职位/组织）。不猜测不可见信息。

第 1 步 — 创建沟通风格侧写：分析：语气（直接 vs 圆融，正式 vs 轻松）、长度（短 / 中 / 长）、结构（条列 vs 叙事，标题，编号，先问结论）、技术深度（低 / 中 / 高）、典型需求（决策 / 脉络 / 风险 / 行动优先）、常用语句或模式（3-5 个短例）。输出：风格摘要（最多 6 条）、Do / Don't 规则（最多 8 条）、信心评级：高 / 中 / 低 + 原因。

第 2 步 — 改写文本：产生 3 个版本：短版（≤6行）、标准版（≤150字）、负责人/决策版（一个画面：所需决策、理由、风险、后续步骤）。规则：保留事实，不添加细节；保持我的声音专业（不加俚语或奇特用法）；信心评级低时：询问 2 个偏好（语气 + 长度），仍生成最佳猜测草稿。加上：建议主旨行（如为邮件）+ 1 个解除进度卡点的追问问题。
📋 Prompt Template (English)
RECIPIENT: [Name / Person] INPUT TEXT: [PASTE YOUR EMAIL / TEXT / SUMMARY HERE] Goal: Rewrite the input text so it matches the recipient's communication style and role context (tech / non-tech), while staying professional, accurate, and fact-based. Sources (Work mode only): Use Outlook emails and Teams chats with the recipient from the last 90 days, plus visible profile signals (role/org). Do not guess beyond what's visible. Step 1 - Build a Communication Style Profile. Analyze: Tone, Length, Structure, Technical depth, Typical asks, Common phrases or patterns (3-5 short examples). Output: Style summary (max 6 bullets), Do / Don't rules (max 8), Confidence: HIGH / MED / LOW + reason. Step 2 - Rewrite the Text. Produce 3 versions: Short (≤ 6 lines), Standard (≤ 150 words), Exec / Decision view (one screen: decision needed, rationale, risks, next steps). Rules: Preserve facts; don't invent details. Keep my voice professional. If confidence is LOW: ask for 2 preferences and still generate a best-guess draft. Add: Suggested subject line (if email) + 1 follow-up question to unblock progress.
💡 三个典型使用场景
1
📋 先贴收件人姓名再跑
在 [收件人] 填入真实的姓名，Copilot 才能访问对应的交互纪录。
2
🎯 选择最符合场景的版本
负责人用「负责人决策版」、同事或合作伙伴用「标准版」，场合对了效果最好。
3
🔄 信心低时先确认偏好
若 Copilot 回报信心评级「低」，多花 30 秒回答 2 个偏好问题，改写质量差很多。
