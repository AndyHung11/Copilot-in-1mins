🎁 你會得到什麼？
📋 可呈報的專案脈動
進度、決策、阻礙、風險、里程碑——一份結構完整、適合 Steering 的更新。

每個缺口或 🔴 都附一封可貼上 Outlook 的信：收件人、主旨、內文。
🚀 直接複製這段 Prompt（繁中版）
打開 Copilot Chat (Web 或 Work)，把下面整段貼上去就能跑。
幫我為專案「[專案名稱]」產出最近 7 天的 Status Update。請使用我的 Email、Teams 對話、會議記錄、近期檔案編輯為來源。請依以下結構產出：1) 執行摘要（3 句），2) 自上次更新以來的進度（條列，含負責人），3) 已做出的決策，4) 待處理的風險與阻礙，並提出建議的因應方式，5) 本週需要做出的決策以及需誰拍板，6) 接下來 14 天的關鍵里程碑。每一條都請標註信心程度（🟢 已確認、🟡 推論、🔴 待驗證），推論項目請註明來源。針對每一個 🔴 項目或關鍵資訊缺失之處，請幫我寫一封簡短、有禮貌的追問信給對應的負責人，索取所需的具體資訊。每封追問信請以可直接複製的區塊呈現，含收件人、建議主旨、完整內文，讓我可以貼進 Outlook、自行檢視後寄出。
📋 Prompt Template (English)
Generate a status updatefor the project "[PROJECT NAME]" covering the last 7 days. Use my emails, Teams chats, meeting notes, and recent file edits as sources. Structure the update as: 1) Executive summary (3 sentences), 2) Progress since last update (bullet list with owners), 3) Decisions made, 4) Open risks and blockers with proposed mitigations, 5) Decisions needed this week and from whom, 6) Upcoming milestones in the next 14 days. Mark each item with a confidence indicator (🟢 confirmed, 🟡 inferred, 🔴 needs validation) and cite the source for inferred items. For every 🔴 item or any point where critical information is missing, write a short, polite follow-up email I can send to the relevant owner asking for the specific input needed. Provide each follow-up as a ready-to-copy block with the recipient, suggested subject line, and full email body, so I can paste it into Outlook and send it myself.
💡 三個情境式玩法
1
🚀 在 Copilot Chat (Work) 貼上
打開 Copilot Chat – Work 並貼上完整 Prompt，讓它掃描你的 M365 訊號。
2
✏️ 換掉 [專案名稱]
填上你實際的專案名，Copilot 才能精準鎖定相關訊號。
3
📨 檢視並寄出追問信
把起草好的追問信複製到 Outlook，調整語氣，下班前真的寄出去。
