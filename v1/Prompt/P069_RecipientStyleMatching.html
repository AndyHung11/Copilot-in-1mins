對方是主管還是工程師？讓 Copilot 自動調整你的溝通語氣
Recipient-Adaptive Message Rewriter
✨ 內容對了，語氣不對也白搭——讓 Copilot 幫你改成對方最愛看的版本 🚀
大多數訊息失敗，不是因為內容有問題——而是因為風格不符合閱讀者的期待。主管要的是決策，技術角色要的是精確度，其他人可能需要先有脈絡。

這個 Prompt 讓 Copilot 根據收件人過去 90 天在 Outlook 和 Teams 的真實互動模式，分析其溝通偏好，再把你的文字改寫成最匹配的版本。不是猜測，是基於真實數據的風格適配——讓你每封信、每則訊息都能獲得想要的回應。
  📋 提示詞範本（繁體中文版）
※ 直接複製整段，貼到 Copilot Chat (Work) 即可使用
收件人：[姓名 / 身份]
輸入文字：[在此貼上你的郵件 / 文字 / 摘要]

目標：改寫輸入文字，使其符合收件人的溝通風格與角色情境（技術型 / 非技術型），同時保持專業、準確、基於事實。

資料來源（僅限 Work 模式）：使用過去 90 天與收件人的 Outlook 郵件和 Teams 訊息，以及可見的個人資料訊號（職位/組織）。不猜測不可見資訊。

第 1 步 — 建立溝通風格側寫：分析：語氣（直接 vs 圓融，正式 vs 輕鬆）、長度（短 / 中 / 長）、結構（條列 vs 敘事，標題，編號，先問結論）、技術深度（低 / 中 / 高）、典型需求（決策 / 脈絡 / 風險 / 行動優先）、常用語句或模式（3-5 個短例）。輸出：風格摘要（最多 6 條）、Do / Don't 規則（最多 8 條）、信心評級：高 / 中 / 低 + 原因。

第 2 步 — 改寫文字：產生 3 個版本：短版（≤6行）、標準版（≤150字）、主管/決策版（一個畫面：所需決策、理由、風險、後續步驟）。規則：保留事實，不添加細節；保持我的聲音專業（不加俚語或奇特用法）；信心評級低時：詢問 2 個偏好（語氣 + 長度），仍生成最佳猜測草稿。加上：建議主旨行（如為郵件）+ 1 個解除進度卡點的追問問題。
  📋 Prompt Template (English)
RECIPIENT: [Name / Person] INPUT TEXT: [PASTE YOUR EMAIL / TEXT / SUMMARY HERE] Goal: Rewrite the input text so it matches the recipient's communication style and role context (tech / non-tech), while staying professional, accurate, and fact-based. Sources (Work mode only): Use Outlook emails and Teams chats with the recipient from the last 90 days, plus visible profile signals (role/org). Do not guess beyond what's visible. Step 1 - Build a Communication Style Profile. Analyze: Tone, Length, Structure, Technical depth, Typical asks, Common phrases or patterns (3-5 short examples). Output: Style summary (max 6 bullets), Do / Don't rules (max 8), Confidence: HIGH / MED / LOW + reason. Step 2 - Rewrite the Text. Produce 3 versions: Short (≤ 6 lines), Standard (≤ 150 words), Exec / Decision view (one screen: decision needed, rationale, risks, next steps). Rules: Preserve facts; don't invent details. Keep my voice professional. If confidence is LOW: ask for 2 preferences and still generate a best-guess draft. Add: Suggested subject line (if email) + 1 follow-up question to unblock progress.
💡 使用小撇步
📋 先貼收件人姓名再跑：在 [收件人] 填入真實的姓名，Copilot 才能存取對應的互動紀錄。
🎯 選擇最符合場景的版本：主管用「主管決策版」、同事或合作夥伴用「標準版」，場合對了效果最好。
🔄 信心低時先確認偏好：若 Copilot 回報信心評級「低」，多花 30 秒回答 2 個偏好問題，改寫品質差很多。
  💬 範例提示詞（Example Prompts）
收件人：工程總監 Amit
輸入文字：我們的 API 整合延遲是因為認證模組還在審查，預計下週三可完成。

請分析 Amit 的溝通風格，並給我短版、標準版、主管決策版三個改寫，附上建議主旨行。
英文版：
Recipient: Engineering Director Amit
Input: Our API integration is delayed because the auth module is still in review. Expected completion: next Wednesday.

Analyze Amit's style and give me short, standard, and exec decision versions with a suggested subject line.
