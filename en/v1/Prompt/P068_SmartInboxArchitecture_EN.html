📬 Smart Inbox Architecture
✨ Less noise, more signal 🎯
Most overflowing inboxes don't have an email problem—they have a noise problem. Newsletters, automated alerts, and CC chains drown out the few messages that actually matter.

This prompt asks Copilot to design a tailored Outlook rule architecture for you: which folders to create, which categories to auto-apply, which senders to mute, and which threads to pull straight to the top.

You stop scrolling and start seeing what matters first.
  📋 Prompt Template (English)
Scope & Guardrails: READ-ONLY analysis. Analyze Outlook emails from the last 14-30 days using METADATA FIRST: sender name, domain, subject line patterns, preview line, To/CC usage, distribution lists, noreply/system signals. Email body content is not summarized and is used only if needed to detect explicit action signals. No deleting emails. Design rules only. No execution. STEP 1 - Inbox Pattern Analysis: Identify patterns: newsletters & broadcasts, FYI digests, community posts, system/tool notifications, meeting invites/updates, external emails, action signals. Create a snapshot of top sender domains, recurring subject keywords, email frequency, and signal vs. noise classification. Highlight messages from manager, skip manager, and key collaborators. STEP 2 - Rule Architecture (DESIGN ONLY): Design inbox rules with guardrails: inbox stays default, no deleting, exceptions for manager/skip/key collaborators and action-required emails. Max 10 rules. For each rule provide: name & intent, conditions, actions, risk level, recommended order (1-10). STEP 3 - Folder Blueprint: Propose up to 6 folders with clear intent, plus quick setup steps for New Outlook and Classic Outlook, including dry-run testing advice.
  📋 提示詞範本（繁體中文版）
範圍與護欄：唯讀分析。分析最近 14–30 天的 Outlook 郵件，優先使用元資料：寄件者名稱、網域、主旨模式、預覽文字、To/CC 使用、通訊群組、noreply/系統訊號。不摘要信件內容，僅在需要時用於偵測明確行動訊號（例如 action required、截止日、核准、升報）。不刪郵件。只設計規則，不執行。

第 1 步 — 收件匣模式分析：識別：電子報與廣播、FYI 摘要、社群貼文、系統/工具通知、會議邀請/更新、外部郵件、行動訊號。建立 Top 寄件者網域、重複主旨關鍵字、郵件頻率、Signal vs Noise 分類快照。標出來自主管、越級主管與關鍵合作者的郵件。

第 2 步 — 規則架構（僅設計）：設計有護欄的規則：收件匣維持預設、不刪除、主管/越級/關鍵合作者與 action-required 郵件設例外。最多 10 條規則。每條規則提供：名稱與目的、條件（寄件者/網域/主旨/類型/To-CC）、動作（資料夾/分類/重要性/標已讀）、風險等級、建議順序（1-10）。

第 3 步 — 資料夾藍圖：最多 6 個資料夾，含清楚目的，以及新版 Outlook 和傳統 Outlook 的快速設定步驟，包含 dry-run 測試建議。
